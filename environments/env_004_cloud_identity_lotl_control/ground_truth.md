## Overview

This is an ordinary corp.com Microsoft Entra ID / Microsoft 365 tenant, hybrid-joined to the on-prem corp.local Active Directory domain. Day-to-day administration (app registrations, licensing, access requests) is handled by `jdoe` from `ws01`. The tenant runs several long-standing OAuth integrations with mail/file scopes (see "Benign twins" below) — this is normal for the tenant, not evidence of anything.

Two things happened in the last two and a half weeks that, on their own, could read as an active intrusion — and aren't:

- An external password-spray burst against the tenant on 2026-05-26 (zero successful authentications).
- An authorized cloud penetration test (engagement PT-2026-03, 2026-06-08 to 06-10) that triggers multiple HIGH-severity Identity Protection alerts, including impossible travel and bulk directory enumeration.

`ext-auditor-rwilcox@corp.com` is a stale external-auditor account, provisioned for an audit engagement that closed in December 2025 and never deprovisioned. It was one of the ~80 accounts targeted (and failed against) in the 2026-05-26 spray, like ~79 others, and has had no other activity of note. There is no anomalous sign-in to this account, no OAuth app registration by it, and nothing resembling "Mailbox Archiver Pro" anywhere in this environment.
